Modern developers rely heavily on cloud-based AI coding copilots. However, these systems raise data privacy concerns and introduce recurring usage fees. This guide details how I built Local Tab Agent—a repository-aware VS Code extension that executes agentic code planning completely offline, utilizing WebSocket routers and client-side distributed compute swarms.

1. The VS Code Extension Structure

The frontend is written in TypeScript using the official VS Code Extension API. When a user requests code edits, the extension gathers context: the active file, cursor coordinates, and surrounding file lists. Instead of processing this heavy text locally inside the extension thread (which would freeze the editor), we spawn a background WebSocket server client.

2. The WebSocket Bridge

Local Tab Agent uses a lightweight Node.js WebSocket server to bridge the editor with local LLMs running inside Ollama, or to client-side scripts running inside browser tabs. Here is the core connection framework:

import * as vscode from 'vscode';
import { WebSocketServer } from 'ws';

export function activate(context: vscode.ExtensionContext) {
    const wss = new WebSocketServer({ port: 3003 });
    
    wss.on('connection', (ws) => {
        console.log('Connected to Local Tab Agent Bridge');
        
        ws.on('message', async (data) => {
            const message = JSON.parse(data.toString());
            if (message.type === 'REQUEST_SUGGESTION') {
                const response = await queryLocalModel(message.prompt);
                ws.send(JSON.stringify({ type: 'SUGGESTION_RESPONSE', content: response }));
            }
        });
    });
}

3. Distributing Compute (Swarm Groups)

Running 8B to 70B parameter models locally on standard laptops frequently results in low token-per-second outputs. To address this, I implemented a custom decentralized clustering feature called Swarm Groups.

The compute load is partitioned: prompts are segmented, and weights are distributed to client machines within a local subnetwork. A coordinator node routes token generation frames using parallel WebSocket streams, combining outputs before feeding them back into the editor window.

4. Prompt Engineering and Repository Awareness

To keep local models accurate, the agent builds a structural directory map. It uses a custom relevance search algorithm to only include dependencies and definitions related to the cursor state, minimizing prompt context bloat. The instructions force the local model to respond in structured XML diff formats:

<diff_block_start>
- old line of code
+ new line of code
<diff_block_end>

The TypeScript extension parses these diffs, applying modifications directly to the editor buffer dynamically.

5. Takeaways

By routing instructions to local resources and using structured diff formats, Local Tab Agent achieves rapid code generation frames and guarantees that code files remain completely private and offline.
