Prosthetics have evolved from inert structures into intelligent interfaces that mirror biological intention. In this article, I discuss the engineering behind an Electromyography (EMG) muscle-controlled mechanical bionic hand, detailing analog amplification, digital filtering, and servo coordination loops.

1. The Biosignal Capture Problem

Skeletal muscle contractions generate microvolt-level potential differences (EMG signals) on the surface of the skin. Measuring these signals is complex because they are obscured by 50Hz/60Hz electromagnetic noise and skin impedance. The signal pipeline requires:

  • Differential Input: Electrodes placed on target muscles capture potentials, reference electrodes ground the signal.
  • Instrumentation Amplifier: A high Common-Mode Rejection Ratio (CMRR) amplifier to reject environmental noise and boost microvolts to millivolts.
  • Bandpass Filter: Restricts passing frequencies strictly to the 20Hz - 500Hz band.
  • Rectification: A full-wave active rectifier to flip negative cycles positive.

2. Real-Time Envelope Extraction (Atmega/Arduino)

Once raw signals are amplified and rectified, the microcontroller ADC samples the analog wave at 1kHz. We extract a smoothed envelope representing muscle contraction force using a digital low-pass filter (Exponential Moving Average):

#include <Servo.h>

const int emgPin = A0;
const int servoCount = 5;
Servo fingers[servoCount];

float emgRaw = 0;
float emgEnvelope = 0;
const float filterAlpha = 0.08; // Cutoff coefficient
const int threshold = 180;      // Activation threshold

void setup() {
  pinMode(emgPin, INPUT);
  fingers[0].attach(3); // Thumb
  fingers[1].attach(5); // Index
  fingers[2].attach(6); // Middle
  fingers[3].attach(9); // Ring
  fingers[4].attach(10); // Pinky
}

3. Moving Average & Actuation Loop

Inside the continuous execution loop, raw analog signals are read, the envelope is updated, and the servo values are adjusted proportionally. When the envelope value crosses the calibrated resting threshold, the fingers close into a grip configuration.

void loop() {
  int sensorValue = analogRead(emgPin);
  
  // Full-wave rectification (centering and absolute value)
  float rectified = abs(sensorValue - 512);
  
  // Exponential Moving Average filter
  emgEnvelope = (filterAlpha * rectified) + ((1.0 - filterAlpha) * emgEnvelope);
  
  if (emgEnvelope > threshold) {
    // Map signal force directly to finger servo angles (close hand)
    int angle = map(emgEnvelope, threshold, 500, 0, 180);
    angle = constrain(angle, 0, 180);
    for(int i=0; i<servoCount; i++) {
      fingers[i].write(angle);
    }
  } else {
    // Hand open state
    for(int i=0; i<servoCount; i++) {
      fingers[i].write(0);
    }
  }
  delay(10); // 100Hz loop execution rate
}

4. Prototyping and Solid Mechanics

The robotic hand chassis was modelled in SolidWorks and 3D printed in PLA. Individual fingers are tension-driven using nylon threads routing back to metal-geared high-torque servos located in the forearm, mimicking actual human tendon motion layout.

5. Clinical Takeaways

Baseline calibration is highly subject to electrode placement drift, sweating, and individual muscle fatigue. Practical systems require dynamic threshold adjustments to normalize resting metrics during operation.
