Voice-driven smart assistants have evolved from rigid command matching into natural, context-aware conversational partners. In this article, I cover the software and hardware architecture of my personal AI assistant, highlighting real-time audio transcription, LLM orchestration, and BLE hardware control loops.

1. Pipeline Architecture

The assistant operates locally in Python, routing requests through a multi-threaded PyQt6 GUI. The audio processing pipeline flows as follows:

  1. Voice Activity Detection (VAD): Continuous audio capture analyzes ambient decibel levels to detect when a user begins speaking.
  2. Speech-to-Text (STT): Offline STT engine transcribes raw vocal waveforms into text strings.
  3. LLM Inference: The transcribed prompt is forwarded to the LLM (Gemini API with fallback local Llama 3 models) along with local system context.
  4. Action Dispatcher: The LLM response is parsed for structural commands (e.g. JSON strings), routing actions to BLE drivers or operating system APIs.

2. Core Python Implementation

We use Python and the Gemini SDK. The assistant features tool-calling so it can interact with the physical world via BLE calls:

import google.generativeai as genai
import json

genai.configure(api_key="YOUR_GEMINI_API_KEY")

# Define structural functions the LLM can call
def control_smart_home_light(state: str):
    """Turns the local desk light on or off. State must be 'ON' or 'OFF'."""
    print(f"Sending BLE command to desk lights: {state}")
    # Raw BLE client calls route here
    return f"Lights set to {state}"

model = genai.GenerativeModel(
    model_name="gemini-1.5-flash",
    tools=[control_smart_home_light]
)

chat = model.start_chat(enable_automatic_function_calling=True)
response = chat.send_message("Please turn on the desk lights for me.")
print(response.text)

3. The Hardware Bridge (Bluetooth Low Energy)

Desk systems and appliances are fitted with STM32 microcontrollers configured as BLE peripherals. When a tool-call is generated, Python utilizes the bleak library to establish connection frames, writing values to target characteristics:

import asyncio
from bleak import BleakClient

DEVICE_MAC = "XX:XX:XX:XX:XX:XX"
CHARACTERISTIC_UUID = "00002a56-0000-1000-8000-00805f9b34fb"

async def send_ble_command(command: str):
    async with BleakClient(DEVICE_MAC) as client:
        payload = command.encode('utf-8')
        await client.write_gatt_char(CHARACTERISTIC_UUID, payload)
        print("BLE Write Successful")

4. Bilingual Processing

The LLM system prompt enforces bilingual operations. Pre-processing scripts parse whether user queries are in English or Arabic, dynamically adjusting speech synthesis parameters (TTS) to provide natural spoken responses in either language.

5. Actionable Outcomes

By shifting parsing from hardcoded regular expressions to semantic tool-calling in LLMs, the voice assistant handles fuzzy commands (e.g., "I can't see anything, help" resolves to calling control_smart_home_light("ON")) with high stability.
