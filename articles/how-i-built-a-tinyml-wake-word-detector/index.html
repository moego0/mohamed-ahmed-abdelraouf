Speech processing has traditionally been the domain of power-hungry cloud servers. However, deploying wake-word detection directly on-device (at the extreme edge) is crucial for privacy, latency, and power conservation. This guide explains how I built and deployed a custom keyword spotting system running under 100KB RAM constraints.

1. The Audio Signal Pipeline

Raw audio streams cannot be directly inputted into a lightweight neural network due to high dimensionality. Instead, we extract Mel-Frequency Cepstral Coefficients (MFCCs). The audio is captured in a 1-second sliding buffer (typically sampled at 16kHz, mono, 16-bit PCM).

Mathematical Extraction Process:
1. Frame the signal into 20ms chunks with a 10ms stride.
2. Apply a Hanning window to prevent spectral leakage.
3. Compute the Discrete Fourier Transform (DFT) to map time to frequency.
4. Pass the power spectrum through a triangular Mel-filterbank to mimic human ear perception.
5. Apply Discrete Cosine Transform (DCT) to get cepstral coefficients.

2. Model Architecture

We train a 2D Convolutional Neural Network (CNN) in Python using TensorFlow and Keras. The input dimension is [49, 10, 1] (representing 49 frames of 10 MFCC coefficients each).

import tensorflow as tf
from tensorflow.keras import layers, models

def build_kws_model(input_shape=(49, 10, 1), num_classes=4):
    model = models.Sequential([
        layers.Input(shape=input_shape),
        layers.Conv2D(32, (3, 3), activation='relu', padding='same'),
        layers.MaxPooling2D((2, 2)),
        layers.Dropout(0.2),
        layers.Conv2D(64, (3, 3), activation='relu', padding='same'),
        layers.MaxPooling2D((2, 2)),
        layers.Flatten(),
        layers.Dense(32, activation='relu'),
        layers.Dropout(0.3),
        layers.Dense(num_classes, activation='softmax')
    ])
    return model

3. Post-Training Quantization (INT8)

Standard microcontrollers do not have hardware Floating Point Units (FPUs) that efficiently process 32-bit floats. We convert the trained float32 model weights and activation outputs to 8-bit integers (INT8) using representative datasets for calibration:

converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
converter.representative_dataset = representative_data_gen
converter.target_spec.supported_ops = [tf.lite.OpsSet.TFLITE_BUILTINS_INT8]
converter.inference_input_type = tf.int8
converter.inference_output_type = tf.int8
tflite_quant_model = converter.convert()

This post-training quantization compressed our model size from 340KB down to a mere 78KB, matching the internal cache layout of STM32 microcontrollers.

4. Edge Deployment using TFLite Micro

Once converted, the model is exported as a static C header byte array and compiled with C++ code utilizing TensorFlow Lite for Microcontrollers. The raw ADC buffer loads audio values, feeds them into the MFCC operator, calls interpreter->Invoke(), and triggers pin controls depending on threshold outputs.

5. Key Takeaways

  • Audio data augmentation (adding kitchen noise, street hums, and pitch fluctuations) is necessary to avoid overfitting on small datasets.
  • Representational dataset calibration must reflect actual environmental soundscapes to maintain high target accuracy in the quantized model.
